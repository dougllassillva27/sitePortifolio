BARBERFLOW — PASSO A PASSO PARA ENVIAR NA IDE/IA
Timestamp: 06/05/2026 21:01

OBJETIVO
Criar o BarberFlow, um sistema de agendamentos para barbearia usado como projeto de portfólio.

ESCOPO TRAVADO DO MVP
- Sistema de agendamentos, não landing page simples.
- 2 barbeiros.
- Cliente escolhe o barbeiro.
- Cliente escolhe serviço, data e horário.
- Cada serviço possui duração própria.
- Agendamento nasce como pendente.
- Admin aceita, cancela, edita, conclui e cria encaixe.
- Cliente pode cancelar e remarcar usando token/link.
- Serviços possuem preço.
- Sem pagamento online no MVP.
- Sem múltiplas unidades.
- Sem múltiplas barbearias.
- Sem WhatsApp automático real.
- WhatsApp no MVP = link manual pré-preenchido.
- Google Calendar será integrado usando minha conta de teste.
- Cliente pode informar e-mail opcional para convite do Google Calendar.
- Admin único.
- Backup obrigatório.
- Supabase como banco, auth, realtime e edge functions.
- Netlify como hospedagem do frontend.

STACK OBRIGATÓRIA
Frontend:
- React
- Vite
- TypeScript
- Tailwind CSS

Backend/Banco:
- Supabase Postgres
- Supabase Auth
- Supabase RLS
- Supabase Realtime
- Supabase Edge Functions

Hospedagem:
- Netlify

Agenda:
- Google Calendar API

WhatsApp:
- Link manual pré-preenchido no MVP
- WhatsApp Cloud API fica fora do MVP

VERSIONAMENTO OBRIGATÓRIO
- Copiar/criar pasta versionamento/ na raiz.
- Manter versionamento/versionador.js.
- package.json deve usar:
  "build": "vite build && node versionamento/versionador.js"
- Proibido adicionar ?v= manualmente nos assets.
- O versionador injeta cache busting no build.

COMO COMEÇAR
1. Execute o arquivo criar-barberflow-estrutura.bat em uma pasta vazia.
2. Entre na pasta criada:
   cd barberflow
3. Instale dependências:
   npm install
4. Rode testes:
   npm run test
5. Rode build:
   npm run build
6. Rode local:
   npm run dev
7. Abra:
   http://localhost:5173

REGRAS DE NEGÓCIO CRÍTICAS
- Status inicial: pendente.
- Aceite admin revalida conflito.
- Cancelamento pelo cliente usa token.
- Remarcação pelo cliente usa token.
- Cliente não precisa login.
- Admin precisa login.
- WhatsApp é manual por link.
- Evento Google só ao aceitar.
- Cliente só recebe convite Google se informar e-mail.
- Encaixe é criação manual do admin.
- Feriado/dia bloqueado impede agendamento.
